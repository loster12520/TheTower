# TheTower

这是聆听的毕设项目。

## 关键词

```text
浏览器RPA系统,可视化流程编排,ReactFlow,Playwright,低代码/无代码
```

## 项目简介

本研究设计并实现一个基于Playwright与ReactFlow的低代码浏览器RPA系统。前端通过ReactFlow实现可视化流程编排，后端利用Playwright完成浏览器自动化操作。重点解决传统RPA技术门槛高的问题，为用户提供轻量级的网页自动化解决方案，实现数据录入、信息抓取等重复性任务的自动化执行。

## 核心内容与可行性

本系统采用前后端分离架构，技术选型成熟、稳定，具备高度可行性：

1. 前端（ReactFlow）：作为可视化画布，负责工作流的编排与展示。ReactFlow是专为节点式应用设计的成熟库，能完美支撑拖拽、连线、配置等交互需求，实现直观的流程设计。
2. 后端（Kotlin + Ktor + Playwright）：作为流程执行引擎。Kotlin + Ktor作为一个广泛应用于安卓应用的后端实现方案，很适用于对于开发速度有较高要求的轻量化项目；同时由于采用Kotlin而非弱类型语言，兼具性能和强类型，更有利于深入开发。Playwright是现代、强大的浏览器自动化库，其跨浏览器支持和智能等待机制能确保自动化流程的稳定、可靠执行。
3. 通信与数据：前后端通过API进行数据交换，前端将画布结构转换为步骤序列传给后端，后端执行并实时返回状态，实现执行过程的可视化监控。

## 创新点与价值

- 技术创新：将Playwright的自动化能力与ReactFlow的可视化编排相结合，构建一个集易用性与强大功能于一体的RPA工具。
- 应用价值：为个人用户及中小企业提供了一个轻量级、低成本的自动化解决方案，摆脱了对昂贵商业RPA软件的依赖。
